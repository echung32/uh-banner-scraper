### Possible follow-up: don't mark partial/error terms as synced

`syncTerm` calls `markTermSynced(..., status, ...)` on both the `partial` and `error` paths
([sync.ts:144](../../web/src/lib/ingest/sync.ts#L144), [:154](../../web/src/lib/ingest/sync.ts#L154)),
setting `last_synced_at` regardless. A throttled/partial term is then indistinguishable from a
complete one to the read path (which only checks `last_synced_at IS NULL`). Worth considering:
only set `last_synced_at` on `ok`, or have the dynamic-path gate also exclude
`last_sync_status != 'ok'`, so partial terms self-heal instead of locking in missing data.
